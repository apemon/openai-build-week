# Gate the Codex Brain harness behind local evaluation

Spec Grill V3 will ship the Bounded Async Interview against the existing authoritative GPT-5.6 Brain adapter while including experimental Brain harnesses behind a server-only mode selector that defaults to `one_shot`. The comparable modes are the current `one_shot` Brain, a `responses_native` analyst/critic/synthesis harness with one bounded repair, and a fresh local `codex_ephemeral` candidate. The Product Manager may enable an experimental mode for local evaluation, but it cannot become the default or a production claim until it materially wins the frozen bake-off and passes the existing schema, semantic, privacy, cancellation, retention, deployment, request-ID, base-revision, complete-snapshot, and `store: false` boundaries; this keeps V3 independently deliverable without treating an unevaluated harness or coding-agent runtime as product authority. Any invented-decision, authority, provenance, privacy, or security violation automatically disqualifies a candidate; otherwise it must win at least 60% of non-tied blind Product Manager comparisons against the current one-shot Brain across at least 24 frozen sessions for question quality and Specification completeness, with end-to-end p95 latency no worse than twice the current Brain on matched sessions and environment. Tokens and estimated cost are recorded when available for hackathon comparison, but cost is informational rather than a promotion gate; spend remains controlled through the dedicated capped OpenAI project and the disabled-by-default server flag. Every `codex_ephemeral` run uses a fresh empty temporary directory, read-only sandbox, ephemeral session, strict environment allowlist, no repository files, no persistent Codex state, and no arbitrary MCP servers; its only input is the same confirmed snapshot and schema supplied to other candidates. Exploratory local runs may additionally use separately enabled, controlled read-only public search, excluding authenticated or private sources and side effects; each authoritative Brain request permits at most five search queries and five opened public sources, and search cannot answer stakeholder decisions or bypass PM confirmation. Search findings remain proposed evidence rather than Confirmed Input. Harness selection and public-search authorization use separate server-only controls that both default to the safe baseline. Before a search-enabled run, the local evaluation surface discloses provider processing and requires explicit acknowledgement; raw queries and results are never logged or checkpointed. The ordinary Live route rejects `codex_ephemeral`, local experimental UI and Markdown provenance disclose the selected adapter and search state without presenting the output as ordinary Live, and Prepared Demo never displays or incorporates experimental provenance. Scored comparisons freeze the retrieved evidence and provide the identical bundle to every candidate, while live-search behavior is evaluated separately so it cannot bias the harness comparison.
