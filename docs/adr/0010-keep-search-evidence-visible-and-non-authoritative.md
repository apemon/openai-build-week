# Keep search evidence visible and non-authoritative

Experimental Spec Grill V3 Brain harnesses may use controlled public search only in an explicitly acknowledged local evaluation run, with separate server-only authorization and a hard maximum of five queries and five opened public sources per authoritative request. Every search-informed recommendation or proposed Specification Item must expose External Evidence with source title, URL, and retrieval time. Search evidence can support only `proposed` content. Product Manager confirmation may authorize the underlying product decision, but it does not confirm a separate external factual claim or transform search evidence into Confirmed Input; search can never independently create `confirmed` or `derived` content. Raw queries, results, and page content are never logged or checkpointed. Markdown export retains evidence reference IDs on affected items and adds an `External Evidence` appendix with each source's title, URL, retrieval time, and informed item IDs, while continuing to exclude Decision Tray work. This preserves traceability and PM authority while allowing current factual research, at the cost of extending validated provenance contracts, presentation, and export rather than hiding retrieval inside model reasoning.
