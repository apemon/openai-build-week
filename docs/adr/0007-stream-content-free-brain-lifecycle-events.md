# Stream content-free Brain lifecycle events with the authoritative response

Spec Grill V3 will replace the Brain route's single terminal JSON body with one same-origin streamed response containing validated, content-free Brain Lifecycle Events followed by the final validated Brain response or typed terminal error. Events are bound to the request ID, action ID, cancellation epoch, and base revision and carry only an attempt number, globally monotonic action sequence, observation time, and allowlisted observable status; they exclude provider response IDs, prompts, answers, summaries, transcripts, Specifications, raw errors, validation details, credentials, and model-authored content. Keeping lifecycle and result delivery on the same request preserves the stateless, browser-authoritative boundary and avoids a separately persisted polling job. A provider-attempt terminal event closes only that attempt and may be followed by validation or the single repair; exactly one final result/error envelope closes the action. Clients reject mismatched, duplicate, out-of-order, post-cancellation, and post-action-terminal events. A malformed or content-bearing envelope compromises and terminates the stream rather than being ignored while processing continues. Persistent Brain Status starts one action-level elapsed timer when the application accepts the PM action and does not reset it for provider queuing, processing, validation, or the single repair attempt. Ten seconds without a valid lifecycle event changes the status to `Needs attention`, stops its active animation, and reports that execution state is unverifiable without claiming provider execution stopped. A broken stream immediately reports `Connection interrupted · Brain state unknown`, stops active animation, and attempts best-effort abort/cancellation without silently restarting authoritative work; only explicit Retry creates a fresh request ID and cancellation epoch, and late output from the old request is rejected locally.
