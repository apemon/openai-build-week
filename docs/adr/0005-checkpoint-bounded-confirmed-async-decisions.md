# Checkpoint bounded confirmed asynchronous decisions

Spec Grill V3 may include at most three individually Product Manager-confirmed queued asynchronous entries—Decision Summaries or permitted deferrals—in the existing expiring per-tab checkpoint so a reload does not silently discard approved wording. Each entry contains only its summary or deferral wording; stable job, exchange, permit, window, roadmap-item, and confirmed-turn IDs; approval revision and dependency version; confirmation record/time; and minimal source kind. The checkpoint may also retain a content-free adaptive-window tuple containing the last three eligible decision-summary outcomes, current application cap, and singleton recovery streak. Drafts, corrections awaiting submission, clarification turns, transcripts, raw speech, unused permits, Not Applied wording, lifecycle events, provider state/IDs, and search content remain excluded. Restored entries become `Awaiting dependency check` and can proceed only after the PM explicitly selects `Revalidate restored decisions` and a fresh authoritative Brain revalidation succeeds; they never auto-submit from reload. Reset, explicit exit, final abandonment, and expiry clear them with the rest of the checkpoint.
